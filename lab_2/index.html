Abstract

This project provides an overview of the process of developing a network and security plan for a small office environment. The focus is on building a basic network diagram, implementing security protocols, and establishing an incident response plan.

Background

The small office in this project consists of four employees, but they might hire another person down the road as they grow. They would like a shared network for their internet services, file sharing, and one office printer. So far, they have not had any incidents. Since they believe they will grow in the next few years, they wanted something manageable but could also frow. The project s intended to address these gaps by providing easy-to-follow documentation that any employee, regardless of technical background, can understand and implement. However, at this time the office manager Sam will be responsible. It should be noted that Sam is also an IT student. This documentation is designed to reduce the complexity of network management and introduce a policy to follow if there is an incident.

Network Diagram Creation

The first step in this project was to create a network diagram to map out all devices and connections within the office. The diagram was created using Lucidchart which is an online diagramming tool. Lucidchart. Allowed for easy visualization of the physical and logical structure of the network. (LucidChart) The objective was to make the diagram simple enough for non-technical staff to understand it while still providing enough detail for Sam to manage their network.

Key Components of the Network Diagram:

  • Workstations for four employees (John, Roberto, Mary, and Sam)
  • Network Printer for shared office printing
  • Wireless Access Point for wireless connectivity
  • Managed Switch for network segmentation
  • Small Web Server managed by Amazon Web Services (AWS) for hosting resources
  • Router to connect the office network to the internet
  • Simple Server with local storage synced to AWS incremental backup

These components are sufficient to get the small office operational. The diagram clearly identifies each component and its use on the network. This provides a clear understanding of the physical and logical structure.

Security Protocol Development

After the network diagram was completed, the next step was to develop basic security protocols to protect the network. These security protocols include password management guidelines, device access policies, and rules for sharing resources. The primary focus was on keeping the security measures straightforward and manageable for a small office without a dedicated IT department.

Security Measures Include:

  • Regular Password Changes: Enforcing a 60-day password change policy through Active Directory using open Group Policy Management Console (GPMC). How to configure a password policy in AD
  • Windows Defender: Use Group Policy settings to configure and manage Microsoft Defender Antivirus. Microsoft Defender setup guide
  • Active Directory and SCCM: Implementing a simple Active Directory (AD) with System Center Configuration Manager (SCCM) for centralized managment. Installing Active Directory and Creating Domain
  • Network Segmentation: Using a managed switch to segment the network and limit lateral movement in case of a breach. This set up is discussed under Network Security Implementation
  • Backup Strategy Performing a small incremental backup every Saturday night, with backups saved for redundancy using Amazon Web Services (AWS) for a small monthly fee. This is also done locally on Simple Server. Backup, restore, migrate, and copy Group Policy Objects
    • Restrict Access: Limiting access to certain parts of the network and avoiding the use of shared credentials.
  • Wireless Security: Configured WPA3 and hidden SSID to protect network access. WPA3 explained

Acceptable Use Policy

An Acceptable Use Policy (AUP) was also developed to provide clear guidelines modeled after FR Secure Acceptable Use Policy Template (FRSecure. 2024). This describes how the office network and devices should be used. The policy for this small office is customized specifically for this environment and includes the following points:

  • Authorized Use Only: The office network and devices are t be used for business purposes only. Personal activities, such as streaming entertainment or accessing social media unrelated to work, are prohibited.
  • No Unauthorized Devices: Employees are not allowed to connect any personal devices (BYOD- Bring Your Own Device) to the work network without explicit authorization. This policy helps prevent unauthorized access and minimizes security risks.
  • Data Protection: Employees must not share sensitive business information without proper authorization. Information, when allowed, should only be shared through secure channels.
  • Internet Use: Access to unauthorized or potentially harmful websites is prohibited. Employees are expected to use the internet in a way that is appropriate and maintains the integrity of the office network.

All employes were required to review and acknowledge the AUP. This will help maintain a secure work environment by ensuring that everyone understands their responsibilities regarding network use

Network Security Implementation

When implementing the network security industry best practices were followed to secure the small office environment. The main goals were to strike a balance between secure and manageable by non-technical staff.

Steps Taken Include:

  • Router Security Secured the router with strong administrative credentials and configured to auto-update the firmware. Set Up DHCP. Dynamic Host Configuration Protocol. This will help have scalability in place for future growth.
  • Firewall Configuration: Configured Firewall using Windows Firewall Rules (Matarazzo, P. & Pamnani, V. 2024) Windows Firewall Rules
  • Network Segmentation Used a managed switch to segment the network. Each port was configured to a specific device. This kept the server separate and would limit lateral movement in case of a breach
  • Switch Model: Netgear GS308E An inexpensive managed switch was selected GS308E_8 Port GB Ethernet Plus Switch
  • Antivirus Protection: Configured Windows Defender on all workstations with regularly scheduled scans to ensure ongoing protection. *Windows Defender is updated through Windows Security System Updates.
  • Backup Strategy Cloud Backups are managed by Amazon Web Services (AWS). On-premises backups are stored on the root of C on Simple Server.
  • Note for BU Strategy Having two backups, one on-prem and the other in the cloud will provide added data protection.

Incident Response Plan:

An incident response plan was also created to guide office staff in the event of a security breach or a network issue. This plan was modeled after FRSecure and then customized to mt the needs of this small office. This plan outlines the steps that should be taken if an incident occurs. The plan was kept simple and easy to follow. The goal of the incident response plan is to minimize downtime and reduce data loss.

  • Step 1: Identify and Document A. Identify the nature of the issue (examples may be unauthorized access, malware infection, disaster). B. Document what has been affected.
  • Step 2: Disconnect Affected Devices Disconnect devices from the network to prevent further spread of the issue.
  • Step 3: Notify the Appropriate Person Contact Sam or IT Dept. Provide Documentation
  • Step 4: Assess the Cause Work with Sam/IT to determine the cause of the incident.
  • Step 5: Restore Systems If necessary, restore systems from Simple Server or Restore systems using AWS.

This documentation provides the basic tools needed to maintain an office network.
